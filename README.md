# 이 프로그램은 피보나치 수열을 이용하여 gcd(F(n), F(n-1)) 값을 계산하는 프로그램이다.
#GCD2
#include "GCD2.h"

int gcd(int a, int b)
{
    if (a < 0)
        a = -a;

    if (b < 0)
        b = -b;

    if (a == 0 && b == 0)
        return 0;

    int result = 1;

    for (int i = 1; i <= a && i <= b; i++)
    {
        if (a % i == 0 && b % i == 0)
        {
            result = i;
        }
    }

    return result;
}

# main.c
#include <stdio.h>
#include "GCD2.h"


int fibo(int n) // 재귀 피보나치
{
    if (n <= 1)
        return n;

    return fibo(n - 1) + fibo(n - 2);
}

int main()
{
    for (int n = 5; n <= 40; n++)
    {
        int a = fibo(n);
        int b = fibo(n - 1);
        int g = gcd(a, b);

        printf("n=%d F(n)=%d F(n-1)=%d gcd=%d\n", n, a, b, g);
    }

    return 0;
}

# fibo(피보나치) 시간복잡도
재귀 함수로 만든 피보나치는 다음 계산식을 가진다.
T(n) = T(n-1) + T(n-2) + O(1)  호출마다 두개의 함수가 다시 호출되므로 수가 지수적으로 증가한다. 따라서 시간 복잡도는 O(2ⁿ)이다.

# GCD 시간복잡도
GCD 알고리즘은 for(i = 1; i <= min(a,b); i++) 알고리즘을 사용한다.
따라서 반복 횟수는  O(min(a,b)) 만큼 수행된다.

# gcd(F(n), F(n-1))의 시간복잡도
피보나치 수열은 F(n) ≈ 1.618ⁿ 처럼 증가한다.
min(a,b) = F(n-1)
이므로 GCD의 시간복잡도는 O(1.618ⁿ)정도로 증가한다

# gcd(F(n), F(n-1)) = 1 이 나오는 이유
연속된 피보나치 수는 항상 서로소이다. 즉 gcd(F(n), F(n-1)) = 1 이 된다.
이유는 피보나치 수는 F(n) = F(n-1) + F(n-2) 이다.또한 GCD에는 다음 성질이 있다. gcd(a+b, b) = gcd(a, b)이를 이용하면gcd(F(n), F(n-1))= gcd(F(n-1)+F(n-2), F(n-1)) = gcd(F(n-2), F(n-1))처럼 계속 더 작은 피보나치 수로 줄어든다.이 과정을 반복하면 결국 gcd(F(2), F(1))이 된다.피보나치 시작 값은 F(2) = 1, F(1) = 1 이므로 gcd(1,1) = 1이다.
따라서 연속된 피보나치 수의 최대공약수는 항상 1이다.

# cpu 및 메모리 사용량 
n =40으로 프로그래밍 하였을때 fibo 함수가 cpu 사용량 95.56%를 차지하고 gcd 함수는 4.25%를 차지하는걸로 보아 재귀 방식으로 구현된 fibo함수가 계산을 여러번 반복하기 때문에 함수 호출이 많아 cpu 사용량 대부분이 fibo함수에서 발생하는것을 확인했다.

메모리 사용량은 801kb 수준으로 프로그램 실행 동안 메모리 사용량이 일정하게 유지되는것으로 보아 프로그램이 동적 메모리를 사용하지 않고  몇 개의 변수만 사용하여 실행되었기 때문인것으로 확인했다.

# 결론
재귀 fibo(피보나치)는 O(2ⁿ)의 시간복잡도를 가지므로n이 증가할수록 CPU 사용량이 크게 증가한다.
또한 GCD 는 O(min(a,b))의 시간복잡도를 가지며 피보나치 수를 입력으로 사용할 경우 O(1.618ⁿ) 정도로 증가한다. 반면 메모리 사용량은 동적 메모리를 사용하지 않기 때문에 사용량이 거의 일정하다.
